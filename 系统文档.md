# 智能教学系统 - 系统文档

## 一、系统概述

### 1.1 系统定位

智能教学系统是一个基于AI的智能题目生成与管理平台，旨在帮助教育工作者高效地创建、管理和审核考试题目。系统通过集成Kimi AI大模型，结合知识库和自定义规则，能够生成高质量、符合教学标准的题目。

### 1.2 核心价值

- **智能生成**：AI自动生成题目，大幅提升命题效率
- **知识驱动**：基于上传的教材、课件等知识库内容生成题目
- **规则定制**：支持自定义命题规则，可从现有试卷中学习风格
- **质量可控**：每道题目包含设计依据、难度理由、干扰项分析
- **流程完善**：生成-审核-采纳-入库的完整工作流

## 三、功能模块

### 3.1 题库管理

| 功能 | 描述 |
|------|------|
| 题目列表 | 分页展示、多条件筛选（题型/难度/状态/知识点） |
| 题目创建 | 手动创建题目，支持LaTeX公式 |
| 题目编辑 | 修改题目内容、选项、答案、解析 |
| 题目审核 | 审核通过/拒绝，记录审核人 |
| 批量操作 | 批量审核、批量删除、批量设为草稿 |
| 统计分析 | 题库数量、题型分布、难度分布统计 |

### 3.2 AI题目生成

| 功能 | 描述 |
|------|------|
| 知识范围选择 | 支持知识点ID、分类、关键词三种方式 |
| 题型配置 | 单选、多选、判断、填空、主观题 |
| 难度配置 | 简单/中等/困难的比例分配 |
| 规则选择 | 选择自定义命题规则 |
| 高级设置 | 干扰项策略、特殊要求 |
| 结果审核 | 查看生成题目，编辑修改 |
| 采纳入库 | 批量采纳为正式题目 |

### 3.3 知识库管理

| 功能 | 描述 |
|------|------|
| 文档上传 | 支持PDF、Word、Markdown、TXT格式 |
| 文档解析 | 自动提取内容、分块存储 |
| 分类管理 | 按章节/主题自动分类 |
| 知识检索 | 按分类、关键词搜索知识内容 |
| 统计信息 | 文档数量、分块数量、字符统计 |

### 3.4 规则管理

| 功能 | 描述 |
|------|------|
| 默认规则 | 系统内置通用命题规则 |
| 自定义规则 | 手动创建学科/场景特定规则 |
| 试卷分析 | 上传试卷，AI分析提取命题风格 |
| 规则应用 | 生成题目时选择应用规则 |

### 3.5 数据分析

| 功能 | 描述 |
|------|------|
| 题库统计 | 总数、各状态数量统计 |
| 题型分布 | 饼图展示各题型占比 |
| 难度分布 | 柱状图展示各难度占比 |
| AI日志 | 查看AI调用记录和详情 |

## 四、API接口

### 4.1 题目管理接口 (`/api`)

| 路由 | 方法 | 功能 |
|------|------|------|
| `/questions` | GET | 分页查询题目列表 |
| `/questions/{id}` | GET | 获取单个题目详情 |
| `/questions` | POST | 创建新题目 |
| `/questions/{id}` | PUT | 更新题目 |
| `/questions/{id}` | DELETE | 删除题目 |
| `/questions/{id}/review` | POST | 审核题目 |
| `/questions/batch-review` | POST | 批量审核 |
| `/questions/batch-draft` | POST | 批量设为草稿 |
| `/questions/batch-delete` | POST | 批量删除 |
| `/stats` | GET | 获取统计数据 |
| `/knowledge-points` | GET | 获取知识点列表 |
| `/tags` | GET | 获取标签列表 |

### 4.2 AI生成接口 (`/api/ai`)

| 路由 | 方法 | 功能 |
|------|------|------|
| `/generate` | POST | 生成题目 |
| `/tasks/{task_id}/questions` | GET | 获取生成任务结果 |
| `/tasks/{task_id}/adopt` | POST | 采纳生成的题目 |
| `/questions/{id}/toggle-draft` | POST | 切换草稿状态 |
| `/questions/{id}/toggle-discard` | POST | 切换丢弃状态 |
| `/questions/{id}` | PUT | 更新生成的题目 |

### 4.3 知识库接口 (`/api/knowledge`)

| 路由 | 方法 | 功能 |
|------|------|------|
| `/documents` | GET | 获取文档列表 |
| `/upload` | POST | 上传文档 |
| `/documents/{id}` | DELETE | 删除文档 |
| `/categories` | GET | 获取知识分类 |
| `/search` | POST | 搜索知识内容 |
| `/chunks/{category}` | GET | 按分类获取知识块 |
| `/stats` | GET | 获取知识库统计 |

### 4.4 规则管理接口 (`/api/rule`)

| 路由 | 方法 | 功能 |
|------|------|------|
| `/rules` | GET | 获取规则列表 |
| `/rules` | POST | 创建规则 |
| `/rules/{id}` | GET | 获取规则详情 |
| `/rules/{id}` | PUT | 更新规则 |
| `/rules/{id}` | DELETE | 删除规则 |
| `/rules/init-default` | POST | 初始化默认规则 |
| `/rules/analyze` | POST | 分析试卷生成规则 |

```
你是一位资深的教育测量与评价专家，拥有20年的考试命题经验。你擅长：
1. 根据布鲁姆认知分类设计不同层次的题目
2. 设计科学合理的干扰项，能够有效区分不同水平的学生
3. 编写清晰、准确、有教育价值的题目
4. 确保题目内容与知识点紧密相关

请严格按照JSON格式返回结果，确保题目质量达到专业考试标准。
```

### 7.1 AI题目生成流程

```
┌─────────────────────────────────────────────────────────────┐
│                    AI题目生成流程                            │
└─────────────────────────────────────────────────────────────┘

1. 用户发起生成请求
   ├── 选择知识范围（知识点ID/分类/关键词）
   ├── 选择题型和数量
   ├── 配置难度分布
   └── 可选：选择自定义规则

2. 后端处理
   ├── 创建GenerationTask记录
   ├── 检索相关知识库内容
   │   ├── 按ID检索
   │   ├── 按分类检索
   │   └── 按关键词搜索
   ├── 构建Prompt
   │   ├── 加载默认规则
   │   ├── 加载自定义规则（如有）
   │   ├── 注入知识库内容
   │   └── 组装完整提示词
   └── 调用Kimi API

3. AI响应处理
   ├── 解析JSON响应
   │   ├── 提取题目数组
   │   ├── LaTeX公式处理
   │   └── 字段验证补全
   ├── 保存GeneratedQuestion记录
   ├── 记录AICallLog日志
   └── 返回生成结果

4. 用户审核
   ├── 查看生成题目
   ├── 编辑/修改题目
   ├── 标记草稿/丢弃
   └── 批量采纳入库

5. 入库处理
   ├── 转换为正式Question
   ├── 创建Option选项
   ├── 关联知识点
   └── 删除GeneratedQuestion
```

### 7.2 知识库处理流程

```
┌─────────────────────────────────────────────────────────────┐
│                    知识库处理流程                            │
└─────────────────────────────────────────────────────────────┘

1. 文档上传
   ├── 接收文件
   ├── 验证格式（PDF/Word/MD/TXT）
   └── 存储到knowledge_base/raw/

2. 文档解析
   ├── PDF解析：提取文本内容
   ├── Word解析：提取段落和标题
   ├── Markdown解析：识别标题层级
   └── TXT解析：按段落分割

3. 内容分块
   ├── 按章节/段落分块
   ├── 提取标题和关键词
   ├── 计算字符数
   └── 存储KnowledgeChunk

4. 知识检索
   ├── 按分类检索：返回指定分类的所有分块
   ├── 按关键词搜索：模糊匹配内容
   └── 字符数限制：控制返回内容总量
```

### 7.3 规则管理流程

```
┌─────────────────────────────────────────────────────────────┐
│                    规则管理流程                              │
└─────────────────────────────────────────────────────────────┘

1. 默认规则
   ├── 系统内置通用命题规则
   ├── 包含角色设定、核心原则等
   └── 不可删除/修改

2. 自定义规则
   ├── 手动创建
   │   ├── 设置角色定位
   │   ├── 定义核心原则
   │   ├── 配置工作流程
   │   └── 设置干扰项策略
   └── AI分析生成
       ├── 上传试卷文件
       ├── AI分析命题风格
       └── 提取学科特化规则

3. 规则应用
   ├── 生成题目时选择规则
   ├── 默认规则 + 自定义规则合并
   └── 冲突时以自定义规则为准
```

## 十一、使用指南

### 11.1 快速开始

1. **配置环境**
   ```bash
   # 安装后端依赖
   cd backend
   pip install -r requirements.txt
   
   # 安装前端依赖
   cd frontend
   npm install
   ```

2. **初始化数据库**
   ```bash
   cd backend
   python init_db.py
   ```

3. **启动服务**
   ```bash
   # 启动后端
   cd backend
   python run.py
   
   # 启动前端
   cd frontend
   npm run serve
   ```

### 11.2 基本使用流程

1. **上传知识库**
   - 进入知识库管理
   - 上传教材、课件等文档
   - 等待解析完成

2. **创建规则（可选）**
   - 进入规则管理
   - 创建自定义规则或分析试卷
   - 保存规则

3. **生成题目**
   - 进入AI生成页面
   - 选择知识范围
   - 配置题型和难度
   - 选择规则（可选）
   - 开始生成

4. **审核采纳**
   - 查看生成的题目
   - 编辑修改
   - 批量采纳入库

5. **题库管理**
   - 进入题库管理
   - 审核题目
   - 导出使用
